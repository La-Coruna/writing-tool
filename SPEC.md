# Writing Tool Spec

## 1. 목적
Cloudscape 디자인 톤(밝은 블루/그레이, 카드 기반, 명확한 정보 계층)을 참고한 단일 페이지 웹 앱을 만든다.

앱은 다음 2가지 핵심 기능을 제공한다.
1. 글자수 세기
2. 두 글의 차이 비교

## 2. 범위
### 포함
- 순수 프론트엔드(HTML/CSS/JavaScript)로 동작
- 한국어 텍스트 입력/비교 지원
- 반응형 레이아웃(데스크톱 + 모바일)

### 제외
- 로그인/저장/동기화
- 서버 API
- 파일 업로드

## 3. 화면 구조
1. 상단 헤더
- 서비스명과 짧은 설명

2. 글자수 세기 카드
- 입력 textarea 1개
- 입력 textarea는 내용 길이에 맞춰 세로로 자동 확장
- 집계 결과 2개:
  - 공백 포함 글자수
  - 공백 미포함 글자수
- 옵션:
  - 괄호 안 내용 제외(체크박스)

3. 차이 비교 카드
- 좌/우 입력 textarea 2개
- 좌/우 각각 글자수 2개 표시:
  - 공백 포함 글자수
  - 공백 미포함 글자수
- 결과 패널 2개(좌측 원문, 우측 비교문)
- git diff 유사 시각화:
  - 삭제: 좌측 빨간 배경
  - 추가: 우측 초록 배경
  - 수정: 좌/우 모두 강조(좌 빨강, 우 초록)

## 4. 기능 요구사항
### 4.1 글자수 세기
- 입력이 변경될 때마다 즉시 집계 갱신
- 입력 textarea 값과 `괄호 안 내용 제외` 옵션 상태를 브라우저 로컬 저장소에 저장하고, 새로고침 후 복원
- 입력 textarea는 스크롤 대신 높이를 자동 조절해 전체 내용을 바로 볼 수 있어야 함
- 한국어/이모지 포함 문자열을 그래프림 단위로 카운트
- 공백 포함 글자수: 전체 그래프림 개수
- 공백 미포함 글자수: 유니코드 공백(`\s`) 제거 후 그래프림 개수
- `괄호 안 내용 제외` 옵션이 켜지면 다음 괄호 쌍 내부 텍스트를 집계에서 제거:
  - `()`, `[]`, `{}`, `（）`
- 중첩 괄호 지원
- 괄호 짝이 맞지 않는 경우:
  - 열린 괄호 뒤 내용은 닫힐 때까지 제외 처리
  - 단독 닫는 괄호는 일반 문자로 취급

### 4.2 두 글 차이 비교
- 줄 단위가 아닌 토큰 단위 비교
- 좌/우 입력 textarea 값을 브라우저 로컬 저장소에 저장하고, 새로고침 후 복원
- 좌/우 입력 각각에 대해 글자수 집계 제공
  - 공백 포함 글자수
  - 공백 미포함 글자수
  - 집계는 4.1과 동일하게 그래프림 기준으로 계산하되, 괄호 제외 옵션은 적용하지 않음
- 토큰 기준:
  - `Intl.Segmenter('ko', { granularity: 'word' })` 우선 사용
  - 미지원 환경은 정규식 기반 분해 fallback
- LCS(Longest Common Subsequence) 기반 diff 계산
- 비동등 구간 처리:
  - 삭제만 존재: 좌측 빨간 하이라이트
  - 추가만 존재: 우측 초록 하이라이트
  - 삭제+추가가 같은 비동등 구간에 함께 존재: 수정으로 간주, 좌/우 동시 하이라이트
- 입력 변경 시 diff 결과와 글자수 모두 자동 갱신

## 5. UI/디자인 요구사항
- Cloudscape 참고 스타일:
  - 높은 가독성, 밝은 배경, 차분한 블루 계열 포인트
  - 섹션 카드, 부드러운 경계선, 명확한 타이포 계층
- 시맨틱 색상:
  - 삭제(빨강), 추가(초록), 일반(중립)
- 모바일(좁은 뷰포트)에서는 좌/우 패널을 세로로 스택

## 6. 비기능 요구사항
- 외부 빌드 도구 없이 로컬에서 바로 실행 가능
- 최신 브라우저(Chrome/Edge/Safari/Firefox 최신 버전) 기준 동작
- 긴 텍스트 입력에서도 UI가 멈추지 않도록 불필요한 렌더링 최소화

## 7. 산출물
- `index.html`: 화면 구조
- `styles.css`: 테마 및 반응형 스타일
- `app.js`: 글자수 계산 + diff 로직
- `SPEC.md`: 본 문서
